1 load bios(hardware information)
2 read MBR's config to find out the OS
3 load the kernel of ths OS
4 init process starts....
5 execute /etc/rc.d/sysinit
6 start other modules (etc/modules.conf)
7 execute the run level scripts
8 execute /etc/rc.d/rc.local
9 execute /bin/login
10 shell started...

mysql show warnings;
可以显示上一次查询的warnings。

select * from t_pandora_statistics where mvalue = 161189566907393966;
我知道了这个问题出现的根源了。因为 mvalue 是varchar(32)的类型，我在查询的时候，用的变量是一个longint的类型，这个时候就汇报这个 warning。

Warning | 1292 | Truncated incorrect DOUBLE value: '12998e8c703749642' 
报告的地方有很多，我只是选择了其中的一个。解决办法是将sql写成这样。
select * from t_pandora_statistics where mvalue = '161189566907393966';
这样就不会报这个错误了。

刚才比较了一下两种写法的效率，我发现不设置返回类型的做法确实在效率上会低很多，所以在写程序
的时候一定要注意一点，不要随便就select * 了，这样是很愚蠢的做法。应该需要什么拿什么，而不是
不管有没有必要，就 select *.性能商的差别可以达到1倍。
